# Moving24 — лендинг (редизайн)

Статический одностраничник. Тестовая публикация: **https://dennymansa.github.io/moving24-landing/**

Источник дизайна: `photo2.html`. Здесь `index.html` — его копия для деплоя.

---

## Что уже работает

- Полный фронтенд (как на боевом moving24.ee по виду), адаптив, форма заявки.
- **Трекинг настроен полностью**, но с placeholder-ID (см. `window.M24` в начале `index.html`).
  С placeholder-ID скрипты не грузятся (чистая консоль), а события `dataLayer` уже шлются:
  - `generate_lead` — при отправке формы
  - `phone_link_klick` — при клике по телефону
  - `email_link_click` — при клике по email
  Это **те же имена событий**, что слушает боевой контейнер GTM — подставишь реальный ID, и теги Ads-команды заработают без изменений с их стороны.
- **Consent Mode v2** — баннер согласия (по умолчанию deny, как требует Google для ЕС).
- **Форма с фото**: фотографии сжимаются прямо в браузере перед отправкой. Сейчас демо-режим
  (`FORM_ENDPOINT` пуст) — чтобы письма реально уходили, см. шаг 2.

## Шаг 1 — подставить реальные ID трекинга

В `index.html` найди объект `window.M24` (вверху, в `<head>`) и замени:

| Поле | Placeholder | Реальное значение (с боевого moving24.ee) |
|---|---|---|
| `GTM` | `GTM-XXXXXXX` | `GTM-TSG2CVLC` |
| `AW` | `AW-XXXXXXXXXX` | `AW-16489866360` |
| `GA4` | `G-XXXXXXXXXX` | `G-7X1TS2BVF6` *(на боевом настроен ВНУТРИ GTM — код менять не нужно)* |
| `CONV_FORM` | `AW-XXXXXXXXXX/FORM_LABEL` | `AW-16489866360/uyGYCLT3uagZEPjI_bY9` |
| `CONV_PHONE` | `AW-XXXXXXXXXX/PHONE_LABEL` | `AW-16489866360/LgJRCOq6nK4ZEPjI_bY9` |

> ⚠️ Подставляя реальный `GTM-TSG2CVLC`, ты подключаешь тестовый сайт к боевой аналитике/рекламе.
> Если не хочешь смешивать тест с реальной статистикой — оставь placeholder или заведи отдельные ID.

Также обнови `<link rel="canonical">` и `og:url` на финальный домен.

## Шаг 2 — включить реальную отправку формы (с фото)

Нужно одно из двух. В `window.M24.FORM_ENDPOINT` вписать URL приёмника.

### Вариант A — Basin (без кода, проще всего)
1. Зарегистрируйся на https://usebasin.com (free: 100 заявок/мес, загрузка файлов до 100 МБ).
2. Создай форму, скопируй её endpoint вида `https://usebasin.com/f/xxxxxxxx`.
3. Впиши его в `FORM_ENDPOINT`. Готово — заявки с фото будут приходить на твой email.

### Вариант B — Cloudflare Pages Function (включена в этот репозиторий)
Файл `functions/api/lead.js` уже написан — это аналог старого флоу «фото → письмо на Gmail», но без WordPress.
1. Хости сайт на **Cloudflare Pages** (см. шаг 3).
2. В Cloudflare → Settings → Environment variables задай:
   - `RESEND_API_KEY` — ключ с https://resend.com (free: 3000 писем/мес)
   - `LEAD_TO` — `info@moving24.ee`
   - `LEAD_FROM` — отправитель на верифицированном в Resend домене, напр. `Moving24 <leads@moving24.ee>`
3. В `window.M24.FORM_ENDPOINT` впиши `/api/lead`.

> На GitHub Pages вариант B не работает (нет серверных функций) — там форма останется в демо-режиме.

## Шаг 3 — продакшен-хостинг (рекомендация: Cloudflare Pages)

GitHub Pages годится только для теста: нет серверных функций (форма демо), и по правилам GitHub
там нельзя коммерческий сайт. Для продакшена — **Cloudflare Pages**:

- бесплатно, **коммерция разрешена**, **безлимитный трафик** (важно при рекламных всплесках);
- свой домен + бесплатный SSL;
- серверные функции (вариант B формы) на бесплатном тарифе.

Публикация:
1. Зайди на https://dash.cloudflare.com → Workers & Pages → Create → Pages.
2. Либо подключи этот репозиторий (`dennymansa/moving24-landing`), либо перетащи папку `deploy`.
3. Привяжи домен/поддомен, задай env-переменные из шага 2B.

Альтернатива через CLI:
```bash
npx wrangler pages deploy .   # из папки deploy, после входа в свой аккаунт Cloudflare
```
